# LP Investment Memo — Section-by-Section Drafting Rules v3

Status: Production drafting guide aligned with `LP Investment Memo Template Schema v3`  
Intended output language: English  
Primary audience: FM Investments / institutional LP investment professionals and authorized drafting systems  
Purpose: Provide an evidence-first operating method for retrieving, analyzing, organizing, drafting, and reviewing an LP investment recommendation memo.

## 1. Relationship to the Template Schema

The companion `LP Investment Memo Template Schema v3` defines what the memo should cover and the ways in which that coverage may be organized.

These Drafting Rules define how to:

- Identify the correct fund, vehicle, strategy, and source scope.
- Inventory and prioritize source materials.
- Map evidence to memo content modules.
- Select an appropriate memo outline.
- Draft each section from supportable evidence.
- Handle missing, preliminary, conflicting, or excessive information.
- Distinguish facts, representations, observations, calculations, and LP-side assessments.
- Validate the completed memo before human review.

The Template Schema governs substantive coverage. These Drafting Rules govern process and execution. If wording in this guide appears to require a fixed heading or sentence form that the Template Schema treats as flexible, the Template Schema's flexibility controls.

## 2. Non-negotiable operating principles

1. Work on one clearly identified investment opportunity at a time.
2. Establish the target identity before extracting facts.
3. Use only source locations authorized for the assignment.
4. Never use a calibration memo as the factual source for a new opportunity.
5. Never move facts among funds, vintages, strategies, vehicles, investor classes, or GPs.
6. Do not invent missing facts or resolve material uncertainty through guesswork.
7. Preserve the source's legal and economic distinctions.
8. Separate GP claims from independently documented facts and LP-side analysis.
9. Draft from an evidence map, not from memory or superficial file-name matching.
10. Required substantive coverage does not necessarily require a standalone heading.
11. Prefer concise synthesis over document-by-document narration.
12. Escalate material conflicts and gaps for human review.

## 3. Information classification

Classify material statements before drafting.

### 3.1 Documented fact

A fact supported by an authoritative document or reliable current record, such as a legal term, organizational role, dated meeting, or reported figure.

### 3.2 GP representation

A statement, expectation, target, market claim, characterization, or explanation provided by the GP. Attribute or qualify it when a reader could otherwise interpret it as independently established.

### 3.3 FM observation

An observation documented in FM meeting notes, diligence records, or internal analysis.

### 3.4 Calculation

A value derived from sourced inputs. Preserve the inputs, formula, assumptions, units, and relevant dates in the evidence record.

### 3.5 LP-side assessment or inference

An analytical conclusion drawn from evidence, such as fee reasonableness, team capacity, strategic coherence, or an execution dependency. Use measured language and retain the supporting rationale.

Do not silently convert a GP representation into a documented fact or an inference into a fact.

## 4. End-to-end drafting workflow

### Phase 1 — Confirm assignment scope

Obtain or determine:

- Target firm / GP.
- Target fund or economic opportunity.
- Strategy and vintage.
- Proposed legal investment vehicle, if known.
- Relevant FM investing entity, if known.
- Memo as-of date.
- Authorized source folder or source list.
- Whether web or public information is permitted.
- Expected output format and house template.

If the requested target is ambiguous, stop factual drafting and obtain clarification.

### Phase 2 — Create the Target Identity Card

Record:

| Identity field | Confirmed value | Source | Status |
|---|---|---|---|
| Firm / investment manager |  |  | Confirmed / preliminary / missing |
| Target opportunity |  |  |  |
| Commercial fund name |  |  |  |
| Legal fund / investment vehicle |  |  |  |
| GP entity |  |  |  |
| Strategy |  |  |  |
| Vintage |  |  |  |
| Related or parallel vehicles |  |  |  |
| Relevant predecessor funds |  |  |  |
| FM investing entity |  |  |  |
| Memo date |  |  |  |

The Target Identity Card is the primary control against cross-fund contamination. Update it when later authoritative materials refine an initially preliminary value.

### Phase 3 — Inventory the source package

Create a file-level inventory of all accessible materials in scope before relying on individual search results. For each file record, when available:

- Exact filename.
- Folder or location.
- Document type.
- Date and version.
- Fund / vehicle relevance.
- Draft, final, executed, or unclear status.
- Likely memo modules supported.
- Readability or access status.

Classify files into useful source families:

- FM internal deal, relationship, meeting, and approval records.
- Legal and terms.
- DDQ and supplemental diligence responses.
- PPM and fundraising / strategy presentations.
- Firm, organization, team, roster, and biography materials.
- Track record, portfolio, performance, and fund model.
- Value creation, operating resources, and case studies.
- Investment process, governance, and committee materials.
- Emails and follow-up records containing substantive information.
- Other or unclear.

The inventory is a coverage control, not a requirement to copy or deeply read every page of every file. Deep-read the most relevant and authoritative materials, then perform a coverage sweep for overlooked files that may contain missing or conflicting evidence.

### Phase 4 — Establish relevance boundaries

For every potentially useful file, determine whether it relates to:

- The current target fund.
- A predecessor fund.
- A parallel or alternative vehicle.
- A related strategy.
- A co-investment.
- The broader firm only.
- An unrelated fund or GP.

Exclude unrelated materials. Use predecessor or related-strategy material only as clearly identified historical or platform context.

### Phase 5 — Build the Section Evidence Map

Before drafting final prose, map material evidence as follows:

| Memo module / field | Evidence or proposed statement | Classification | Source and locator | As-of date | Confidence | Conflict / follow-up |
|---|---|---|---|---|---|---|

Every material legal term, quantitative claim, role, date, strategy range, performance figure, and LP conclusion should be traceable through this map.

### Phase 6 — Choose the memo architecture

Select the closest structure from the Template Schema:

- Compact.
- Expanded.
- Specialized / multi-fund.
- Another evidence-driven structure.

Use the four calibrated presentation patterns as defaults when they fit. Do not infer facts because a fund resembles a pattern. Combine or depart from patterns when necessary.

### Phase 7 — Draft section by section

Draft from the evidence map. Keep terminology consistent across sections and avoid repeating the same fact unless the repetition serves a clear analytical purpose.

### Phase 8 — Coverage and contradiction review

After the first draft:

1. Revisit the file inventory.
2. Identify expected content that remains unsupported.
3. Search unexamined but likely relevant sources.
4. Reconcile or flag material conflicts.
5. Confirm that optional content is included only when useful.
6. Confirm that combined headings still cover the required substance.

### Phase 9 — Final QA and human handoff

Run the global and section-specific checks in this guide. Deliver the draft with a concise list of unresolved material items rather than hiding uncertainty inside polished prose.

## 5. General source hierarchy and conflict handling

Source authority depends on the statement being drafted. Do not apply one universal hierarchy without regard to subject matter.

### 5.1 Legal and economic terms

Typical priority:

1. Executed or final current-fund legal document.
2. Latest current draft legal document, clearly labeled as draft.
3. Final or latest current-fund term sheet or legal summary.
4. Current PPM.
5. Current DDQ or formal supplemental response.
6. Current fundraising presentation.
7. Meeting notes or email explanation.

Side letters, election forms, fee letters, and negotiated terms may modify the general fund terms and must be considered when applicable to FM.

### 5.2 Team, roles, and governance

Typical priority:

1. Current dated organizational chart, team roster, or formal governance document.
2. Current DDQ and key-person schedule.
3. Current biographies and formal team materials.
4. Recent diligence notes that clarify actual responsibilities or recent changes.
5. Older materials, used only with date and status awareness.

The latest source is not automatically correct if it describes a different strategy or vehicle.

### 5.3 Strategy and portfolio construction

Typical priority:

1. Current PPM and formal current-fund strategy documents.
2. Current DDQ and portfolio construction materials.
3. Current fundraising presentation.
4. Diligence discussions and written follow-ups.
5. Historical portfolio and track-record evidence for comparison, not as a substitute for the stated current strategy.

### 5.4 Meetings and diligence history

Typical priority:

1. Final FM meeting notes or approved internal diligence record.
2. Reliable transcript or meeting recap.
3. Substantive follow-up email corroborating participation and subject matter.
4. Calendar record as a lead only.

A calendar invitation is not proof that every invitee attended.

### 5.5 Firm background

Typical priority:

1. Current DDQ and formal organizational materials.
2. Current PPM or firm presentation.
3. FM relationship records.
4. Authorized public information, if permitted.

Public information should not override current confidential fund or legal materials.

### 5.6 Resolving conflicts

When sources conflict:

1. Confirm they concern the same fund, vehicle, class, period, and definition.
2. Compare authority, finality, date, and scope.
3. Determine whether one source expressly supersedes another.
4. Resolve only when the basis is clear and document the decision in the evidence map.
5. If the conflict is material and unresolved, flag it rather than silently selecting a value.

## 6. Normalization standards

### Names and entities

- Use the full verified legal name where legal precision is required.
- Use a consistent approved short name after first reference.
- Preserve entity suffixes in legal fields.
- Do not infer an email address from a person's name or company domain.

### Dates

- Use the house style consistently.
- Preserve exact dates when available.
- Do not convert an approximate or expected date into a confirmed date.
- Include as-of dates for figures whose timing affects interpretation.

### Currency and numbers

- Preserve source currency unless a conversion is required and authorized.
- Label conversions and assumptions.
- Distinguish millions from billions and use consistent abbreviations.
- Preserve ranges and qualifiers such as approximately, up to, at least, target, and hard cap.

### Percentages and basis points

- Do not confuse percentage points and basis points.
- Preserve fee bases, thresholds, tiers, and applicable periods.
- Label gross versus net performance and fund-level versus investment-level measures.

### Experience and tenure

- Distinguish total professional or investment experience from years at the firm.
- Distinguish current from former roles.
- Avoid calculating age, tenure, or experience unless the inputs and as-of date are reliable.

### Strategy terminology

- Preserve material distinctions such as control, minority, growth, venture, buyout, structured, primary, secondary, and co-investment.
- Do not describe a target allocation as an actual historical allocation.

## 7. Rules for Document Title

### Objective

Identify the investment recommendation clearly and consistently.

### Preferred sources

- FM deal-intake or approval record.
- Current fund documents.
- User-confirmed opportunity name.

### Drafting rules

- Use the approved house title format from the Template Schema.
- Prefer a concise opportunity name in the title.
- Confirm that the title matches the target opportunity, not merely the broader firm.
- If linked funds or vehicles are involved, identify the principal recommendation and explain the relationship elsewhere.

### Missing or conflicting information

- If the commercial opportunity name is unclear, do not finalize the title.
- Do not substitute a legal entity name that changes the apparent economic opportunity without explanation.

### Quality checks

- [ ] Title identifies the correct target opportunity.
- [ ] Fund number, strategy, and geography are correct where included.
- [ ] Title is consistent with Core Investment Information and Fund Terms.

## 8. Rules for Core Investment Information

### Objective

Provide the controlling identification fields for the recommendation.

### Preferred sources

- FM internal deal and relationship records.
- Current term sheet, subscription documents, LPA, or legal summary.
- Current GP contact record.
- User-confirmed memo date and FM investing entity.

### Information to extract

- Firm Name.
- Firm Point of Contact and verified email, when available.
- Investment Opportunity.
- Investment Vehicle Name.
- FM Global Investing Entity.
- Memo Date.

### Matching rules

- Match the firm to the investment manager / GP, not the fund name.
- Match the opportunity to the economic fund or strategy under review.
- Match the investment vehicle to the legal entity through which FM would invest.
- Check whether commercial, legal, parallel, feeder, and alternative-vehicle names differ.

### Drafting rules

- Use concise label-value presentation.
- Preserve full legal names and jurisdictional suffixes where relevant.
- Use `TBD`, `Not yet confirmed`, or another accurate status when a field is genuinely unresolved.
- If multiple entities are involved, add a short explanatory note or related-vehicle subsection rather than compressing them into one ambiguous name.

### Prohibited behaviors

- Do not infer an email address.
- Do not use a predecessor vehicle as the current investment vehicle.
- Do not treat the GP entity and investment vehicle as interchangeable.

### Quality checks

- [ ] Every populated value is supported by a current or authorized internal source.
- [ ] The target fund and legal vehicle are not confused.
- [ ] The FM investing entity has been verified.
- [ ] Unresolved fields are visibly labeled rather than guessed.

## 9. Rules for Onsite Meetings and Subsequent Due Diligence

### Objective

Summarize the chronology and substance of FM's meaningful diligence interactions.

### Preferred sources

1. Final FM meeting notes or internal activity record.
2. Reliable transcripts or recaps.
3. Substantive follow-up emails.
4. Calendar records as leads and date corroboration.

### Information to extract

For each substantive interaction, capture the supported subset of:

- Date.
- Interaction type or descriptive label.
- FM participants.
- GP or other participants and roles.
- In-person, virtual, or other format when useful.
- Main purpose and topics.
- Material update, observation, or follow-up.

### Retrieval and deduplication

- Search meeting notes, transcripts, emails, CRM / activity history, and calendars using the firm, fund, participant, and topic names.
- Group multiple records describing the same event.
- Use the strongest evidence for attendance and content.
- Exclude purely administrative interactions unless they affected diligence.

### Drafting rules

- Present interactions in chronological order unless the house template requires otherwise.
- Use dated entries, short event paragraphs, a chronological narrative, or a combination.
- Use natural verbs that accurately describe the interaction; no canonical sentence pattern is required.
- Include participant names when decision-useful and supported. A group description may be used when individual names are unavailable or unnecessary.
- Summarize the purpose and material topics concisely.
- Avoid overloading the section with every detail from the meeting notes.

### Missing or conflicting information

- If attendance is uncertain, do not convert invitees into attendees.
- If the date is supported but detailed notes are missing, state only the supported purpose or event description.
- If no substantive interactions are located, identify that status for human review rather than inventing a history.

### Prohibited behaviors

- Do not list invitations as completed meetings without corroboration.
- Do not duplicate one event because it appears in multiple records.
- Do not add topics that were planned but not documented as discussed.

### Quality checks

- [ ] Entries are chronological and deduplicated.
- [ ] Dates and participants are supported.
- [ ] Each entry explains why the interaction was relevant.
- [ ] The section is concise and excludes administrative noise.

## 10. Rules for Fund Terms Summary

### Objective

Present the material legal and economic terms of FM's investment clearly, accurately, and in a strategy-appropriate form.

### Preferred sources

Use the legal and economic source hierarchy in Section 5.1. Review side letters, fee letters, election forms, or negotiated terms that may apply specifically to FM.

### Retrieval method

1. Confirm the target fund, vehicle, and investor class.
2. Identify the latest and most authoritative legal / terms documents.
3. Extract the common term rows from the Template Schema.
4. Classify the fund's strategy and structure.
5. Add applicable strategy- or structure-specific rows.
6. Compare duplicate values across sources.
7. Record open or preliminary terms explicitly.

### Information to extract

At minimum, assess whether the following are applicable and available:

- Fund and GP names.
- Strategy, stage, sector, and geography.
- Target size, hard cap, and fundraising timing.
- Management fee during and after the investment period.
- Carry, hurdle, catch-up, and waterfall.
- Investment period and fund term.
- GP commitment.
- Key Man / Key Person provision.
- Fee offsets, expenses, and discounts.
- Leverage / subscription facilities.
- Recycling.
- Co-investment.
- Reasonableness of fees.

Then assess fund-specific additions such as classes, tiers, diversification, allocation, reserves, parallel vehicles, hedging, liquidity, or specialized protections.

### Drafting rules

- Use a structured table.
- Use extra columns or grouped rows when terms vary by class, commitment, closing, or performance tier.
- Preserve rates together with their bases and applicable periods.
- Preserve thresholds, consent standards, exceptions, and extension mechanics when material.
- Summarize long provisions in plain but legally faithful language.
- Keep `Key Man` legal terms separate from Key Person biographies.
- Label preliminary, undisclosed, pending, or unresolved terms accurately.
- Write `Reasonableness of Fees` as a balanced LP assessment only when there is a supportable basis.

### Conflict handling

- Confirm whether apparently conflicting values apply to different classes or periods.
- Prefer executed / final terms over drafts and presentations.
- Do not silently replace an FM-specific negotiated term with the general fund term.
- Escalate material unresolved differences.

### Prohibited behaviors

- Do not report a fee percentage without its base or applicable period when those details are available.
- Do not collapse different investor classes into one blended term.
- Do not present target size as hard cap.
- Do not infer undisclosed terms from a predecessor fund.
- Do not characterize fees as market standard solely because the GP says so.

### Quality checks

- [ ] Fund, vehicle, and class are correct.
- [ ] Fee rates include bases and periods.
- [ ] Target size and hard cap are distinct.
- [ ] Investment period and fund term are distinct.
- [ ] Carry, hurdle, catch-up, and waterfall are internally consistent.
- [ ] FM-specific terms have been considered.
- [ ] Open terms remain clearly open.
- [ ] Narrative references agree with the table.

## 11. Rules for Investment Manager / Firm / Platform Context

### Objective

Explain the GP as an organization and the target strategy's place within the broader platform.

### Preferred sources

- Current DDQ.
- Current organizational and firm overview materials.
- PPM.
- Current firm / fundraising presentation.
- FM relationship records.
- Authorized public information, only when permitted.

### Information to extract

Select the most decision-relevant of:

- Founding, ownership, leadership, and history.
- AUM, professionals, and office footprint with relevant dates.
- Main platforms, strategies, and fund series.
- Relevant experience and investment history.
- Target strategy's organizational position.
- Shared platform resources and infrastructure.
- Material organizational, strategic, governance, or performance evolution.
- Related strategies and allocation or overlap considerations.

### Drafting rules

- Begin with a concise firm overview.
- Move quickly to why the platform matters to the target fund.
- Distinguish dedicated resources from firm-wide totals.
- When related strategies matter, explain their mandates and interaction; use a comparison table if it improves clarity.
- Include prior performance context only when it is relevant, correctly scoped, and clearly dated.

### Prohibited behaviors

- Do not reproduce a marketing-oriented corporate history without underwriting relevance.
- Do not imply that all firm resources are dedicated to the target fund.
- Do not mix AUM, personnel, or performance figures with different as-of dates without explanation.

### Quality checks

- [ ] Firm facts are current or appropriately dated.
- [ ] The target strategy's place in the platform is clear.
- [ ] Dedicated and shared resources are distinguished.
- [ ] Related strategies are discussed only when material.
- [ ] The section is analytical rather than promotional.

## 12. Rules for Team, Governance, and Decision-Making Bodies

### Objective

Explain whether the people and governance structure appear capable of executing the target strategy and how responsibility and authority are allocated.

### Preferred sources

- Current DDQ and team appendix.
- Current organizational chart and roster.
- Governance and committee materials.
- Legal Key Person schedule.
- Current biographies.
- Diligence notes concerning responsibilities, turnover, succession, capacity, incentives, and decision making.

### Information to extract

- Dedicated team size and seniority mix.
- Day-to-day leadership.
- Experience, tenure, continuity, and succession.
- Geographic, sector, stage, and functional coverage.
- Staffing model and resource allocation.
- Broader operating and support resources.
- Material hires, promotions, departures, or reorganizations.
- Relevant committee composition and responsibilities.
- Approval stages, voting mechanics, decision rights, vetoes, and escalation.
- Allocation decisions among related funds or vehicles.
- Monitoring and exit / hold governance, when relevant.

### Drafting rules

- Organize the section around execution responsibility, not a list of names.
- Use prose, subheadings, a leadership table, or a combination.
- Place governance under Team or Investment Process according to readability, but avoid duplication.
- Explain actual decision rights rather than merely naming a committee.
- When team changes are material, explain their effect on capacity, continuity, alignment, or succession.
- Distinguish the dedicated target team from shared platform staff.

### Missing or conflicting information

- If team counts differ, check dates, definitions, offices, strategies, and included seniority levels.
- If formal titles differ from described responsibilities, present the supported distinction or flag it.
- If governance mechanics are not documented, do not infer them from committee names.

### Prohibited behaviors

- Do not add senior firm executives merely because they are prominent.
- Do not equate committee membership with legal Key Person status.
- Do not calculate average experience or tenure from incomplete rosters without disclosure.

### Quality checks

- [ ] Day-to-day responsibility is clear.
- [ ] Dedicated versus shared resources are clear.
- [ ] Governance bodies and decision rights are accurately described.
- [ ] Material team changes and capacity issues are addressed.
- [ ] Content is not duplicated unnecessarily elsewhere.

## 13. Rules for Key Persons

### Objective

Provide decision-useful information about the individuals most relevant to the target fund.

### Preferred sources

- Current biographies.
- DDQ and team roster.
- Organization chart.
- Governance materials.
- LPA or legal Key Person schedule.
- Diligence notes clarifying actual responsibilities.

### Selection rules

Include individuals because they materially affect:

- Target-strategy leadership.
- Investment decisions.
- Sourcing or portfolio oversight.
- Sector, stage, geographic, or operating execution.
- Legal Key Person protection.

Do not use one universal number of biographies. Select enough people to explain responsibility and key-person dependence without turning the section into a firm directory.

### Drafting rules

- Use short biographies, a table, a team graphic plus prose, or a combination.
- For prose, generally move from current role to relevant experience, prior roles, representative responsibilities, and optional education.
- For tables, choose columns that match the evidence; do not force age, tenure, education, or investment examples when they are not useful or verified.
- Use consistent depth across individuals with similar relevance.
- Clearly distinguish legal Key Persons from other important professionals when both are discussed.

### Prohibited behaviors

- Do not infer current titles from older biographies.
- Do not treat education or prestige as a substitute for role relevance.
- Do not list an individual as a legal Key Person without legal support.
- Do not import biographies from another strategy without confirming current involvement.

### Quality checks

- [ ] Every person is relevant to the target strategy.
- [ ] Titles and roles are current.
- [ ] Legal Key Person status is accurately distinguished.
- [ ] Biographies are supported and reasonably consistent in depth.
- [ ] The section complements rather than repeats Team.

## 14. Rules for Strategy

### Objective

Explain what the fund will invest in, why the opportunity may exist, how the strategy is expected to generate returns, and what could impair execution.

### Preferred sources

- Current PPM and formal strategy materials.
- Current DDQ.
- Portfolio-construction model.
- Fundraising presentation.
- Track record and predecessor-fund portfolio materials.
- Diligence notes and written follow-ups.

### Information to extract

Select the supported, material dimensions:

- Stage, transaction type, and ownership orientation.
- Company size and investment / equity check.
- Geography and sectors.
- Portfolio construction, pace, reserves, and concentration.
- Target-company characteristics.
- Sourcing model and access claims.
- Market opportunity and competitive context.
- Underwriting approach and return drivers.
- Co-investment use.
- Return objectives, correctly labeled.
- Material execution, market, concentration, allocation, and strategy risks.
- Relationship to related funds or strategies.

### Pattern selection

Choose the closest presentation approach:

- Traditional sector / geography / size framework.
- Dedicated regional or market-segment strategy.
- Venture / growth market map, thematic roadmap, or platform model.
- Specialized vertical or complex-situation strategy.
- Multi-fund comparison and allocation framework.

Combine or adapt patterns when required by the evidence.

### Drafting rules

- Start with a concise statement of the strategy's central proposition.
- Organize the supporting discussion by the dimensions most important to that strategy.
- Use bullets, subheadings, tables, or visuals when they clarify sectors, criteria, themes, or related strategies.
- Separate target strategy from actual historical execution.
- Attribute GP market statistics, sourcing claims, and forecasts when material.
- Use historical examples as evidence or illustration with fund and time context.
- Include balanced LP considerations, not only opportunity statements.
- Place detailed operating levers in Value Creation unless integrating them improves coherence.

### Prohibited behaviors

- Do not mechanically fill every possible strategy category.
- Do not present target allocation as actual allocation.
- Do not mix investment-level and fund-level return targets.
- Do not use predecessor performance as proof that the current strategy will perform similarly.
- Do not force a buyout framework onto venture, growth, or specialized strategies.

### Quality checks

- [ ] Reader can understand what the fund intends to buy.
- [ ] Size, stage, ownership, geography, and sector descriptions are consistent.
- [ ] Portfolio construction and allocation mechanics are included when material.
- [ ] GP claims are appropriately attributed or qualified.
- [ ] Return measures are correctly labeled.
- [ ] Material LP risks and dependencies are visible.

## 15. Rules for Value Creation / Value-Add Capabilities

### Objective

Explain how the GP expects to support portfolio companies or otherwise create value, and whether the model appears supported by resources, process, and evidence.

### Placement rule

Cover the substance when relevant, but use a standalone heading only when it improves clarity. The content may be integrated into Strategy or Investment Process.

### Preferred sources

- Value-creation and operating-team presentations.
- DDQ operating-resource responses.
- Functional playbooks and portfolio-support materials.
- Portfolio-company case studies.
- KPI, performance bridge, and operating data.
- Diligence discussions with investment and operating professionals.

### Information to extract

- Engagement and ownership model.
- Dedicated and shared operating resources.
- Timing of involvement.
- Relevant value-creation levers.
- Initiative ownership, implementation, measurement, and monitoring.
- Selected examples and dated outcomes.
- Organic / inorganic and GP / market attribution, where available.
- Limitations, failed examples, or execution dependencies.

### Drafting rules

- Describe only the levers relevant to the GP and strategy.
- Explain how capabilities are deployed, not only that they exist.
- Connect resources, process, and examples where the evidence allows.
- Use dated quantitative evidence carefully and describe measurement scope.
- Distinguish GP claims from demonstrated results.
- Acknowledge dependencies such as management cooperation, specialist capacity, integration, or market conditions.

### Prohibited behaviors

- Do not attribute all revenue or EBITDA growth to GP initiatives without evidence.
- Do not treat operating-team headcount as proof of effectiveness.
- Do not select only successful case studies if material contrary evidence is available.
- Do not duplicate Strategy or Investment Process verbatim.

### Quality checks

- [ ] Value-creation substance is covered somewhere in the memo when relevant.
- [ ] Resources and implementation process are distinguished from claims.
- [ ] Examples are dated and appropriately scoped.
- [ ] Attribution limitations are preserved.
- [ ] Placement avoids unnecessary duplication.

## 16. Rules for Investment Process and Decision Making

### Objective

Explain how investments are sourced, evaluated, approved, allocated, supported, and monitored, focusing on real controls and decision rights.

### Preferred sources

- DDQ investment-process responses.
- PPM.
- Governance and committee materials.
- IC process summaries.
- Diligence notes.
- Operating-resource and portfolio-review materials.

### Framework selection

Use one or combine several of the following:

1. **Chronological:** sourcing through post-close monitoring.
2. **Underwriting workstreams:** market, management, revenue, cost, operations, balance sheet, legal, regulatory, technology, and other diligence.
3. **Governance / allocation:** staffing, committee review, voting, decision rights, allocation among funds, and boundary cases.

### Drafting rules

- Select the framework that best explains the actual process.
- Describe approval stages and who has authority at each stage.
- Identify specialist involvement and timing when material.
- Explain challenge, downside testing, independent review, or escalation mechanisms when supported.
- For multi-fund platforms, explain allocation rules and treatment of overlap or boundary cases.
- Describe post-close planning and portfolio monitoring when meaningful.
- Use concrete processes rather than generic labels.

### Prohibited behaviors

- Do not call a process rigorous or disciplined without explaining the relevant controls.
- Do not infer voting or veto mechanics.
- Do not confuse deal-team recommendation with final approval authority.
- Do not force a chronological description when workstreams or allocation mechanics are more material.

### Quality checks

- [ ] The process framework fits the strategy.
- [ ] Decision rights and approval stages are clear.
- [ ] Material diligence workstreams are covered.
- [ ] Allocation and conflict mechanics are covered when relevant.
- [ ] Post-close oversight is included when supported.
- [ ] Generic language has been replaced with specific evidence.

## 17. Rules for Material Changes to Strategy or Process

### Inclusion test

Include a change section only if all of the following are true:

1. A prior approach can be identified reliably.
2. A current approach differs materially.
3. The change matters to underwriting or LP risk.
4. Evidence supports the comparison.

If these conditions are not met, omit the section.

### Preferred sources

- Current and predecessor PPM / DDQ comparisons.
- Current and predecessor portfolio construction and actual portfolio data.
- Governance documents.
- Diligence notes explaining the rationale.
- Examples showing implementation.
- FM internal analysis.

### Drafting structure

Use:

`Prior approach → current approach → reason → implementation evidence → LP implication and remaining risk`

The section may use a concise narrative, a comparison table, or a small number of titled observations.

### Drafting rules

- Separate formal change from gradual evolution or observed drift.
- Explain whether the change is stated, implemented, or only intended.
- Connect the change to prior experience or lessons only when supported.
- Discuss both potential benefit and execution dependency.

### Prohibited behaviors

- Do not manufacture a change narrative because a predecessor fund exists.
- Do not treat renamed terminology as a substantive change without evidence.
- Do not claim successful implementation solely because the GP announced the change.

### Quality checks

- [ ] Prior and current approaches are both supported.
- [ ] Materiality is clear.
- [ ] Rationale and implementation evidence are distinguished.
- [ ] LP implication is balanced.
- [ ] Section is omitted if the inclusion test fails.

## 18. Rules for LP-Side Takeaways and Assessments

### Objective

Translate material evidence into concise implications for an LP reader.

### Appropriate uses

- Fee reasonableness.
- Team continuity or capacity.
- Platform advantage and dependence.
- Strategic differentiation.
- Governance strength or weakness.
- Allocation or conflict risk.
- Value-creation credibility.
- Process improvement and remaining execution risk.

### Drafting rules

- Use selectively, not after every section.
- Ground each takeaway in evidence already presented or mapped.
- State why the issue matters to the LP.
- Distinguish observation from conclusion.
- Use calibrated language such as `appears`, `may`, `supports`, `depends on`, or `remains subject to` when certainty is limited.
- A takeaway may be a bold sentence, short concluding paragraph, table assessment, or embedded observation.

### Prohibited behaviors

- Do not introduce new unsupported facts in a takeaway.
- Do not write promotional conclusions.
- Do not convert one example into a broad conclusion without justification.
- Do not imply final investment approval unless explicitly authorized.

### Quality checks

- [ ] Each takeaway has traceable support.
- [ ] LP relevance is explicit.
- [ ] Language matches the strength of evidence.
- [ ] Takeaways add analysis rather than repeat description.

## 19. Rules for Related Funds, Vehicles, Classes, and Allocation

### Inclusion trigger

Use this module when more than one fund, vehicle, sleeve, class, or adjacent strategy materially affects economics, mandate, staffing, sourcing, allocation, governance, or conflicts.

### Information to extract

- Role of each entity or strategy.
- Relevant legal and commercial names.
- Mandate differences.
- Class-specific economics.
- Shared team and governance.
- Allocation rules and decision makers.
- Overlap and boundary cases.
- Potential conflicts and mitigating controls.

### Drafting rules

- Identify the primary recommendation first.
- Use a comparison table when it improves clarity.
- Keep economics attached to the correct class or vehicle.
- Explain allocation mechanics where they affect access, diversification, or conflicts.
- Place the module near Core Information, Fund Terms, Firm / Platform, Strategy, or Investment Process according to its primary purpose.

### Quality checks

- [ ] Every entity and class has a clear role.
- [ ] Terms are attached to the correct entity or class.
- [ ] Allocation and overlap are explained when material.
- [ ] The reader can identify exactly what FM is being asked to invest in.

## 20. Rules for Open Diligence Items

### Objective

Preserve material uncertainty and support efficient human follow-up.

### When to include

Include open items when they could affect:

- Legal identity or subscription path.
- Economics or investor rights.
- Team, Key Person, or governance assessment.
- Strategy scope or allocation.
- Performance interpretation.
- Value-creation or process credibility.
- The recommendation or required approvals.

### Drafting options

- Accurate status within the relevant field or table row.
- A short note within the relevant section.
- A consolidated open-items list for human review when several material questions remain.

### Drafting rules

- State what is unknown and why it matters.
- Do not overstate that information is unavailable unless the relevant sources were actually reviewed.
- Distinguish `not disclosed`, `not found`, `not yet finalized`, and `conflicting`.
- Avoid allowing a consolidated open-items list to replace transparent status in the memo itself.

## 21. Source-search guidance by content family

### 21.1 Rapid filename and folder triage

Before opening documents in depth, scan the immediate folder structure and file list. Use folder names, filenames, dates, version labels, and target-fund references to route the search efficiently.

Apply the following initial priority, adapting to the actual package naming:

1. **Legal folders and clean/current term summaries or term sheets** for core identifiers, vehicle names, economics, closing information, and key terms.
2. **Clean/current/final/executed LPA and governing documents** for exact entities, fees, carry, term, Key Person provisions, governance, borrowing, recycling, and investor protections.
3. **Team information, team structure, organization charts, biographies, rosters, and Key Person materials** for Team, Governance, and Key Persons.
4. **DDQ, due-diligence files, meeting materials, and FM notes** for Firm, Team, Strategy, Value Creation, Investment Process, changes, and LP-side observations.
5. **PPM, fundraising decks, strategy presentations, portfolio-construction materials, and track-record files** for mandate, sectors, geography, stage, sourcing, portfolio construction, returns, historical execution, and value creation.
6. **Other files** when needed to close an evidence gap, corroborate a material statement, or resolve a conflict.

Use exact target-fund names, recognized abbreviations, strategy names, vintage numbers, legal entities, and relevant participant names to distinguish the current target from other funds in the package.

### 21.2 Files to deprioritize initially

Deprioritize or skip files whose names indicate that they are primarily document comparisons rather than clean source versions, including labels such as:

- `redline` or the informally used label `right line`.
- `blackline`.
- `comparison` or `compare`.
- `marked`, `markup`, or `track changes`.
- `changes`, `diff`, or similar version-comparison wording.

Prefer a clean, current, final, or executed version when one is available. Comparison files may be reviewed later when:

- The assignment requires analysis of changes between versions.
- No clean version can be located.
- A material conflict must be explained.
- The comparison itself provides reliable evidence of a negotiated or revised provision.

When a comparison file must be used, record the limitation and do not silently treat the marked document as the final governing version.

Also deprioritize clearly unrelated funds, duplicate exports, administrative cover files, and obsolete versions after confirming that they are not needed to understand a material change or conflict.

### 21.3 Filename limitations

Filenames and folder labels are routing clues, not evidence. Before using a fact:

- Open the document and verify its content.
- Confirm the target fund, vehicle, class, and effective date.
- Confirm whether the document is clean, draft, final, or executed.
- Verify that the relevant passage applies to the current target rather than another fund in the same document.

Package size does not determine authority. A short executed legal document may be more authoritative than a large presentation, while a highly relevant team appendix may be more useful for Key Persons than a general PPM.

### 21.4 Search concepts by content family

Use file names and folder labels as search leads, not as proof of relevance. Helpful search concepts include:

| Content family | Useful source concepts and search terms |
|---|---|
| Core Information | subscription, vehicle, entity, contact, term sheet, approval, intake |
| Meetings | meeting notes, call, webinar, annual meeting, update, Q&A, transcript, follow-up |
| Terms | LPA, term sheet, legal summary, PPM, fee, carry, hurdle, key person, expenses, co-investment |
| Firm | firm overview, organization, ownership, AUM, offices, platform, history |
| Team | team, roster, organization chart, biographies, leadership, departures, succession, capacity |
| Governance | investment committee, growth committee, voting, approval, allocation, LPAC, portfolio review |
| Strategy | strategy, mandate, portfolio construction, target, sector, geography, stage, sourcing, market map |
| Value Creation | operating team, value creation, portfolio support, playbook, case study, KPI, performance bridge |
| Process | investment process, underwriting, diligence, IC, approval, monitoring, 100-day plan, exit review |
| Changes | predecessor, prior fund, lessons learned, evolution, revised, new process, strategy shift |

Search using the exact target fund name and recognized abbreviations. When a document contains multiple funds, retrieve the relevant section and verify every extracted statement's fund scope.

## 22. Coverage sweep and reverse-check

After the initial draft, conduct a structured reverse-check:

1. List every material statement in the draft.
2. Confirm its evidence-map entry.
3. Identify memo modules with weak or single-source support.
4. Revisit the file inventory for likely overlooked sources.
5. Search authoritative documents using the relevant section concepts.
6. Confirm that a missing fact is truly not located, rather than merely absent from the first search result.
7. Recheck multi-fund documents for scope contamination.

When calibrating methodology against an approved historical memo, the approved memo may be used to identify a topic that should be reverse-mapped to the historical source package. It must not be used as the source of a new fund's facts.

## 23. Final global quality-control checklist

### Scope and identity

- [ ] The Target Identity Card is complete or visibly marks unresolved fields.
- [ ] The draft concerns only the authorized target opportunity.
- [ ] Related and predecessor funds are clearly labeled.
- [ ] The legal investment vehicle and FM investing entity are correct or accurately unresolved.

### Evidence

- [ ] Material statements are traceable through the evidence map.
- [ ] GP representations, FM observations, calculations, and LP assessments are distinguishable where needed.
- [ ] Material conflicts are resolved transparently or escalated.
- [ ] Missing information is not guessed.

### Structure and coverage

- [ ] All Core content is covered.
- [ ] Expected content is covered when relevant and supported.
- [ ] Conditional content appears only when its trigger is met.
- [ ] Combined sections still provide the required substance.
- [ ] No empty or generic filler sections remain.

### Terms and quantitative accuracy

- [ ] Fund terms match the correct vehicle and class.
- [ ] Fees retain bases, periods, tiers, and thresholds.
- [ ] Dates, currencies, units, percentages, and basis points are consistent.
- [ ] Performance figures are dated and labeled gross / net and fund / investment level.
- [ ] Tables agree with narrative.

### Team, strategy, and process

- [ ] Team members are relevant to the target strategy.
- [ ] Legal Key Persons are distinguished from important professionals.
- [ ] Target strategy is distinguished from historical execution.
- [ ] Value-creation claims are distinguished from evidence.
- [ ] Decision rights and allocation mechanics are accurately represented.

### Style

- [ ] Tone is concise, analytical, and LP-oriented.
- [ ] Promotional and unsupported language has been removed.
- [ ] Repetition has been reduced.
- [ ] Uncertainty is preserved.
- [ ] The memo reads as a coherent underwriting document.

## 24. Required handoff package

When the workflow permits, provide the human reviewer with:

1. Clean draft memo.
2. Section Evidence Map.
3. Target Identity Card.
4. Material conflicts and open diligence items.
5. Short QA summary identifying any checks that did not pass.

Do not describe the memo as final or approved unless the authorized human review and approval process has been completed.

## 25. Known limitations

- These rules cannot overcome inaccessible, unreadable, incomplete, or incorrectly scoped source materials.
- File names and search retrieval are leads, not proof that the full source package was reviewed.
- Strategy-specific terms and structures may require additional modules beyond those observed in the calibration set.
- Market-practice judgments, legal interpretation, and investment conclusions may require qualified human review.
- The four calibration patterns should be expanded as additional approved memo / source-package pairs reveal durable new structures.
