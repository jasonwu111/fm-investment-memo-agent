======================================================================

Before beginning the methodology-development workflow, read the two uploaded control documents.

Confirm your understanding by explaining:

1. The exact filename and role of each control document.
2. Which document defines WHAT the final memo should contain.

6. The exact title of the new document you are expected to produce.

- Correctly identifies both files.
- Schema = WHAT; Authoring Brief = HOW.

PROMPT 1 — INVENTORY ACCESSIBLE CALIBRATION MATERIALS
======================================================================

Proceed only with the inventory phase.

Inventory every accessible file in the connected calibration folders. For each item, provide:

- Case ID or proposed case ID.
- Firm and fund, if determinable from the file itself.
- Exact filename.
- Folder path or available location identifier.
- Document type.
- Document date or as-of date.
- Draft, final, approved, duplicate, or unknown status.
- Proposed role: final memo, legal/terms source, GP diligence material, FM note, supporting analysis, or other.
- Readiness/access status.

Then propose a pairing between each approved final memo and its corresponding source package. Explain the evidence supporting every pairing.

Flag:

- Missing expected materials.
- Multiple memo versions.
- Ambiguous final versions.
- Duplicate files.
- Files that appear to belong to another fund.
- Files you cannot open or fully read.

Do not build drafting rules and do not inspect any designated held-out validation case.

ACCEPTANCE CHECKS:
- Complete file-level inventory.
- Pairings use explicit folder/file evidence.
- Drafts and duplicates are flagged.
- No held-out case used.
- No drafting rules produced.

======================================================================

Using the approved Template Schema and completed source-to-section map, draft a standalone document titled:

LP Investment Memo - Section-by-Section Drafting Rules v1

For every memo section and field, include:

2. Required or conditional status.
3. Information to capture.

7. Attribution requirements.
8. Missing-information treatment.

- Cross-fund contamination.
- Invented facts or terms.
- Silent conflict resolution.
- Treating GP marketing claims as FM conclusions.
- Unsupported optional sections.
- Combining distinct legal entities or fund vehicles.

Mark the document "DRAFT — NOT YET VALIDATED." Do not claim it is production-ready.

The rulebook must be reusable for future firms, funds, strategies, vehicles, and folder structures. Do not embed Bain Capital Insurance Fund II, Bessemer Century Fund III, their URLs, folder paths, filenames, people, fund terms, or other case-specific facts as operative rules. If an example is useful, anonymize it as Target Fund, Predecessor Fund, Firm A, Case 01, or another clearly generic placeholder.

Separate:

- General reusable drafting rule.
- Observed calibration pattern.
- Case-specific fact that must not enter the rulebook.

ACCEPTANCE CHECKS:
- Standalone rulebook, not a commentary about the rulebook.
- Covers all required and conditional sections.
- Includes evidence, missing-data, conflict, and QC rules.
- Clearly marked unvalidated.
- Contains no case-specific hard coding in operative rules.

PROMPT 6 — HELD-OUT CASE VALIDATION
======================================================================

The held-out validation case has now been connected. Do not use its approved final memo as an input while producing the simulated memo output.

Using only:

- The Template Schema.
- The draft Section-by-Section Drafting Rules.
- The held-out case source package, excluding its final memo.

perform the following:

1. Build a source inventory for the held-out case.
2. Build an evidence map.
3. Identify missing and conflicting information.
4. Produce a simulated section-by-section memo draft.
5. Cite the supporting source and locator for every material fact.

After completing the simulated output, compare it with the approved held-out final memo. Report every discrepancy in a validation table containing:

- Section or field.
- Simulated output.
- Approved historical output.
- Relevant source evidence.
- Error category.
- Root cause.
- Proposed rule revision.

Use these error categories where applicable:

- Retrieval failure.
- Wrong source selected.
- Extraction error.
- Summarization error.
- Attribution error.
- Unsupported inference.
- Cross-fund contamination.
- Missing required content.
- Incorrect optional-section decision.
- Formatting or tone mismatch.

Do not declare the rules validated merely because the outputs are similar. Focus on factual support and rule compliance.

ACCEPTANCE CHECKS:
- Final memo withheld until after simulated drafting.
- Every discrepancy classified.
- Root causes translated into proposed rule changes.
- Accuracy evaluated separately from stylistic similarity.
