SOURCE-PACKAGE RETRIEVAL GUIDANCE FOR BOTH CASES

1. First inspect folder names and filenames to identify likely fund relevance, document type, date, and version.
2. Prioritize legal-related folders and clean/current summary term sheets or fund-term summaries for core identifiers and fund terms.
3. Prioritize clean/current/executed LPA and related governing documents for exact legal terms.
4. Prioritize team information, organization charts, biographies, and key-person materials for Team and Key Persons.
5. Prioritize due diligence files, DDQs, meeting materials, and FM notes for Firm, Team, Strategy, Value Creation, Investment Process, and changes over time.
6. Use PPMs, fundraising decks, and strategy materials for mandate, portfolio construction, sectors, geography, sourcing, target returns, and value creation.
7. Deprioritize files labeled redline, blackline, comparison, marked, or changes when a clean/current/final/executed version is available. Use comparison files only to understand changes or when no clean version exists, and flag that limitation.
8. Treat filenames as routing clues only. Verify fund name, vehicle, date, version, and content before using a fact.

PAIRING RULE

The Approved Final Memo and Source Package located within the same attached SharePoint case folder should be treated as a proposed pair. The Agent must verify the pairing during inventory and stop for clarification if the content indicates otherwise.
